# Voice & Terminology

The app's interface copy is direct, professional, and specific. This is a working tool for a busy person, not a consumer product trying to delight.

## Tone

- **Professional, not corporate.** "Approve & Publish" not "Looks great! Ship it!"
- **Specific, not vague.** "3 articles in review" not "You have items to check"
- **Businesslike chrome.** The UI elements (buttons, labels, navigation) are straightforward. Save the personality for the chat agent.
- **No emoji in UI.** The chat agent can be warmer, but the interface chrome stays clean.

## Pipeline Terminology

Use editorial terms, not software development terms:
- **Researching** (not "Backlog" or "To Do")
- **Drafting** (not "In Progress")
- **Review** (not "QA" or "Testing")
- **Published** (not "Done" or "Complete")

## Error Messages

Errors should be clear about what happened and what to do next. Never cute, never vague.

- "Research failed. The agent couldn't find enough quality sources for this topic. Try narrowing the focus or adding specific source URLs."
- "Publishing failed. Check that the API endpoint is configured in settings."
- "Draft generation timed out. Try again, or write this one manually."

## Empty States

Empty states are invitations, not error screens:
- Empty pipeline: "No articles in the pipeline. Start one from the backlog, or brainstorm with the agent."
- Empty backlog: "No topics yet. Add one, or ask the agent what's worth writing about this week."
- Empty published: "Nothing published yet. Your first article is waiting."

## Labels and Actions

- "Start Research" (not "Begin" or "Initiate")
- "Send Back" (not "Reject" or "Return to Draft")
- "Approve & Publish" (not "Ship" or "Go Live")
- "Add Topic" (not "Create New" or "New Entry")
- "Revision Notes" (not "Feedback" or "Comments")
